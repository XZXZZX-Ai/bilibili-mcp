## New WBI Test Details

**503 retry test**: Stubs `fetch` to return `Response(503)`. The non-ok response throws `NetworkError` with `statusCode: 503`. `shouldRetry` checks `statusCode` first — 503 is in `retryableStatusCodes` — so four attempts are made. The outer catch preserves `error.statusCode` via `error instanceof NetworkError ? error.statusCode : undefined`. Asserts `statusCode: 503` and 4 fetch calls.

**AbortError test**: Stubs `fetch` to throw `new DOMException("...", "AbortError")`. Not `instanceof TypeError`, so falls through inner catch. `shouldRetry` matches by `error.name`. Four attempts made. Outer catch converts `AbortError` → `TimeoutError`. Asserts `name: "TimeoutError"` and 4 fetch calls.

## Python Test Details

`test_stop_summary.py` imports `matches`, `harness_artifact_reminders`, and `main` directly. Eight deterministic functions:

- `test_matches_exact_file`, `test_matches_prefix` — path matching logic
- `test_codemap_branch`, `test_security_branch` — single-hit reminders
- `test_eval_branch_below_threshold` (2 paths, no eval), `test_eval_branch_at_threshold` (3 paths, triggers eval)
- `test_no_false_positive` — non-harness paths produce no reminders
- `test_main_stdout_is_json_control` — captures stdout, asserts `{"suppressOutput": true}`; calling `main` refreshes only the hook's approved Codex runtime observation files outside the repository

No dependencies beyond stdlib. The direct `main` check refreshes only approved runtime observation files; it performs no repository or Git mutation.
